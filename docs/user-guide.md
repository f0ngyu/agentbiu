# AgentBIU 傻瓜式使用说明

这是一份给普通用户的操作说明。你不需要了解 Cursor、OpenClaw 或命令行开发流程，只需要按照下面步骤操作即可。

## 适用范围

- 系统：macOS
- 网络：BSC
- 用途：在 four.meme 上发币

## 你需要准备什么

- 一个支持 BSC 的浏览器钱包，例如 `MetaMask` 或 `OKX Wallet`
- 少量 `BNB` 作为 gas
- 如果你选择 `USD1 / ASTER / USDC` 这类底池，还需要钱包里提前准备对应币种余额
- 用于发币的图片、名称、符号、描述

## 发布包目录说明

打包后你会得到一个文件夹：`AgentBIU-macos`

里面最重要的文件有：

- `start.command`
  双击即可启动程序并自动打开浏览器
- `stop.command`
  停止本地服务
- `.env.example`
  配置模板
- `用户使用说明.md`
  当前这份说明

## 第一次使用

1. 双击 `start.command`
2. 程序会自动打开浏览器
3. 如果目录里还没有 `.env`，程序会自动从 `.env.example` 复制一份

## 两种使用方式

### 1. 浏览器钱包模式

适合大多数用户。

步骤：

1. 打开页面
2. 选择 `浏览器钱包`
3. 点击连接钱包
4. 切换到 `BSC`
5. 填写发币信息
6. 如果需要，先完成 `8004` 检查和申请
7. 点击发币

注意：

- 如果选择 `BNB` 底池，只需要足够的 `BNB`
- 如果选择 `USD1 / ASTER / USDC` 底池，钱包里必须先有对应币种余额
- 非 `BNB` 底池在发币前可能会先弹一次 `approve` 授权，再弹创建交易

### 2. 私钥模式

适合本地测试或自动化操作。

步骤：

1. 在页面中切换到 `私钥模式`
2. 临时输入私钥，或者在 `.env` 里写入 `PRIVATE_KEY`
3. 填写发币信息
4. 点击发币

安全提醒：

- 不要把 `.env` 发给别人
- 不要把私钥提交到 git 或聊天软件
- 建议使用单独的钱包进行测试

## `.env` 配置

常见配置如下：

```bash
PORT=3000
BSC_RPC_URL=https://bsc-dataseed.binance.org
# PRIVATE_KEY=0xyour_private_key_here
```

说明：

- `PORT`
  本地服务端口，默认 `3000`
- `BSC_RPC_URL`
  可选，不填也能使用默认公共 RPC
- `PRIVATE_KEY`
  仅私钥模式需要

## 发币页面字段说明

- `Token 图片`
  建议使用 `PNG / JPG / WEBP / GIF`
- `Token Name`
  代币名称
- `Ticker Symbol`
  代币简称
- `Description`
  必填
- `Raised Token`
  选择底池币种
- `创建时预买入 XXX`
  创建 token 时顺带打一笔首购资金，填 `0` 表示不预买入
- `8004 Agent 名称`
  用于申请 `8004 NFT`

## 成功后你会看到什么

结果区域会显示：

- `CA 地址`
- 钱包地址
- 创建费
- 交易哈希
- `复制 CA`
- `复制哈希`

## 常见问题

### 1. 提示“交易币种余额不足，请补充后重试”

说明你选择了非 `BNB` 底池，但钱包里没有足够的对应币种余额。

例如：

- 选 `USD1` 底池 -> 需要足够的 `USD1`
- 选 `USDC` 底池 -> 需要足够的 `USDC`

同时仍然要保留少量 `BNB` 作为 gas。

### 2. 只看到交易哈希，没有看到 CA

新版本会优先从链上回执解析 `TokenCreate` 事件来显示 `CA`。  
如果偶发没显示，可以打开 `BscScan` 查看同一笔交易。

### 3. 启动后浏览器没自动打开

手动访问：

- `http://127.0.0.1:3000`

如果你是开发模式，也可能是：

- `http://127.0.0.1:5173`

### 4. 如何停止程序

双击 `stop.command`，或者直接关闭启动它的终端窗口。

## 面向开发者

如果你不是普通用户，而是需要本地开发，请回到仓库根目录执行：

```bash
bun install
bun run dev
```
